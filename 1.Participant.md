#   Participant 


### 1. The Sender
The **Sender** is the entity that initiates the data transfer or triggers the integration process. 
* **Role:** It "pushes" data to SAP CPI or waits for CPI to "pull" data from it.
* **Technical Connection:** You connect the Sender to the **Start Message** event or an **Inbound Adapter** (like HTTPS, IDoc, or SFTP).
* **Example:** A Salesforce system sending an outbound message to SAP CPI to update an order.

### 2. The Receiver
The **Receiver** is the target system where the processed data is ultimately delivered.
* **Role:** It consumes the output of the integration flow.
* **Technical Connection:** You connect the **End Message** event or a **Request-Reply** step to the Receiver using an **Outbound Adapter** (like SOAP, Mail, or OData).
* **Example:** An On-Premise SAP S/4HANA system receiving updated customer details.
